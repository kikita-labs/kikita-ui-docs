  • kuiCard applies visual treatment only. Choose the host element for its native semantics: article/section/aside for static grouped content, button or a for a clickable card.
  • interactive only changes hover and focus-visible styling. It does not add a role, keyboard handler, or click behavior, so the host element must already provide those (a real button or an a with a valid href).
  • Keep a real heading element inside the card for the card's accessible structure instead of relying on visual styling alone.
